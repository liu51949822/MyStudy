# 📖 第 6 课：运行与测试

## 6.1 快速启动

### 第一步：安装依赖

```bash
cd rag-tutorial
npm install
```

这会安装 `package.json` 中的所有依赖：
- `next`、`react`、`react-dom`：Web 框架
- `openai`：DeepSeek API 客户端
- `pg`：PostgreSQL 客户端
- `typescript`、`@types/*`：TypeScript 类型支持

### 第二步：配置环境变量

复制示例文件并填入你的配置：

```bash
cp .env.local.example .env.local
```

编辑 `.env.local`：

```env
DEEPSEEK_API_KEY=sk-your-real-key-here
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/rag_tutorial
```

### 第三步：确保 PostgreSQL 已启动

```bash
# 检查 PostgreSQL 是否在运行
pg_isready

# 如果没在运行，启动它
brew services start postgresql@16
```

### 第四步：创建数据库

```bash
# 创建数据库（如果还没有的话）
createdb rag_tutorial

# 验证连接
psql -d rag_tutorial -c "SELECT 1+1;"
```

### 第五步：启动开发服务器

```bash
npm run dev
```

看到输出：
```
▲ Next.js 14.x
- Local: http://localhost:3000
✓ Ready in 3.2s
```

## 6.2 使用教程

### 步骤 1：存入知识

在浏览器中打开 http://localhost:3000

在"知识入库"区域的文本框中，输入：

```
RAG 是 Retrieval-Augmented Generation 的缩写，中文叫"检索增强生成"。
它是一种让 AI 在回答问题前先去知识库检索相关信息的架构。
RAG 的核心流程：首先把知识库中的文档转换成向量存储起来，
当用户提问时，把问题也转换成向量，然后在知识库中搜索最相似的文档，
最后把检索到的文档作为参考资料提供给 AI，让 AI 基于这些资料来回答问题。

RAG 的主要优势：
1. 提高回答准确性：AI 基于具体资料回答，减少胡编乱造
2. 结合私有知识：可以把公司文档、产品手册等私有知识接入 AI
3. 知识可更新：更新知识库即可更新 AI 的知识，无需重新训练模型
4. 可追溯：AI 的回答可以追溯到具体的参考来源
```

点击"存入知识库"。看到蓝色提示"文本已成功存入知识库"表示成功。

> **可以重复这个过程**，存入更多内容。每次存入一条新记录。

### 步骤 2：上传文件（新增）

你也可以通过上传文件来批量导入知识。支持 `.txt` 和 `.md` 格式。

在"文件上传"区域：

1. 点击"选择文件"，选一个 `.txt` 或 `.md` 文件
2. 点击"上传并入库"
3. 控制台会显示分块和入库的详细日志：
   ```
   ✂️ 正在分块...
      → 共分为 5 个文本块
   🔮 正在处理第 1/5 块...
   🔮 正在处理第 2/5 块...
   ...
   ```

> 文件内容会自动按段落分块后存入知识库，每个块各自生成向量。

### 步骤 3：提问

在"知识问答"区域，输入问题：

```
什么是 RAG？它有什么优点？
```

点击"提问"。等待几秒钟，你会看到 AI 的回答，以及参考来源。

同样可以试试：

```
RAG 的核心流程是什么？
```

```
RAG 和传统 AI 有什么区别？
```

```
法国的首都是什么？
```
（这个问题知识库里没有，AI 会说"资料库中未找到相关信息"）

## 6.3 直接在命令行测试 API

你也可以通过 curl 在命令行测试：

```bash
# 存入知识
curl -X POST http://localhost:3000/api/ingest \
  -H "Content-Type: application/json" \
  -d '{"content": "嵌入向量是把文本转换成数字表示的技术。"}'

# 上传文件
echo "# 测试文档
嵌入向量（Embedding）是 NLP 中的核心概念。
它把文字转换成一串数字，语义相近的文字向量也相近。" > /tmp/test-rag.md

curl -X POST http://localhost:3000/api/upload \
  -F "file=@/tmp/test-rag.md"

# 提问
curl -X POST http://localhost:3000/api/query \
  -H "Content-Type: application/json" \
  -d '{"question": "什么是嵌入向量？"}'
```

## 6.5 常见问题排查

### Q1：页面报 "DEEPSEEK_API_KEY is not set"

**原因**：没有创建 `.env.local` 文件或文件里没有正确填写 API Key。

**解决**：检查 `.env.local` 文件是否存在，内容是否正确。

### Q2：页面报 "connect ECONNREFUSED"

**原因**：PostgreSQL 没有运行。

**解决**：
```bash
brew services start postgresql@16
```

### Q3：页面报 "database 'rag_tutorial' does not exist"

**原因**：数据库还没有创建。

**解决**：
```bash
createdb rag_tutorial
```

### Q4：AI 回答说的是"资料库中未找到相关信息"

**原因**：你提问的内容，知识库里没有相关的文档。

**解决**：先往知识库里存入相关文本，再提问。

### Q5：存入或查询很慢

**原因**：
1. 第一次调用 DeepSeek API 会比较慢（冷启动）
2. 网络延迟（国内访问 DeepSeek 可能较慢）
3. 数据库在本地，首次连接需要建表

**解决**：第二次调用通常会快很多。

### Q6：npm install 报错

**原因**：Node.js 版本太旧。

**解决**：
```bash
node --version  # 需要 >= 18
```
如果版本太低，去 https://nodejs.org 下载最新 LTS 版本。

## 6.6 课后练习

### 练习 1：扩展知识库

存入更多不同类型的文本，测试 AI 能否准确回答：
- 一段产品说明
- 一段技术教程
- 一段公司制度

### 练习 2：查看参考资料

每次提问后，展开"查看参考来源"，看看 AI 是根据哪些资料回答的。

### 练习 3：修改 System Prompt

在 `lib/embeddings.ts` 中找到 system message，修改它的内容，观察 AI 回答风格的变化。

比如改成英文或更正式的语气：
```typescript
content: "You are a helpful assistant. Answer based on the context provided.",
```

### 练习 4：调整相似度参数

在 `lib/rag.ts` 的 `query` 函数中，把 `searchSimilarDocuments` 的 limit 改为 1 或 5，
看看回答有什么不同。

## 6.7 进阶方向

学会了这个最简单的 RAG，你可以进一步学习：

1. **文档拆分（Chunking）**：长文档先切分成小段再存入
2. **多轮对话**：结合聊天历史
3. **文件上传**：支持上传 PDF、Word 等文件
4. **混合搜索**：结合关键词搜索和向量搜索
5. **Reranking**：对检索结果进行重排序
6. **多种模型**：支持 Claude、Gemini 等

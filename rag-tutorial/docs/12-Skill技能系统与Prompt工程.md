# 📖 第 12 课：Skill 技能系统与 Prompt 工程

## 12.1 什么是 Skill 系统？

Skill = 技能。每个 Skill 是一个独立的 AI 能力模块。

```
┌──────────────────────────────────────┐
│        技能注册中心 (registry)        │
│                                      │
│  ┌──────┐  ┌──────┐  ┌──────┐      │
│  │ 问答  │  │ 总结  │  │ 翻译  │      │
│  │ 技能  │  │ 技能  │  │ 技能  │      │
│  └──────┘  └──────┘  └──────┘      │
│                                      │
│  ┌──────┐  ┌──────┐                │
│  │ 分析  │  │ 其他  │  ← 可以扩展    │
│  │ 技能  │  │ 技能  │                │
│  └──────┘  └──────┘                │
└──────────────────────────────────────┘
```

## 12.2 Skill 接口

```typescript
interface Skill {
  id: string;              // 唯一标识，如 "qa", "summarize"
  name: string;            // 显示名称，如 "📖 知识问答"
  description: string;     // 功能说明
  execute(input): Promise<output>;  // 执行函数
}
```

## 12.3 内置技能

| 技能 | ID | 功能 |
|------|-----|------|
| 📖 知识问答 | qa | 基于知识库回答（默认） |
| 📝 内容总结 | summarize | 提炼内容要点 |
| 🌐 翻译 | translate | 翻译内容 |
| 🔍 深度分析 | analyze | 结构化分析 |

## 12.4 Prompt 模板引擎

```typescript
// 模板
"你是一个{{role}}。请{{action}}。"

// 渲染
renderTemplate(template, { role: "助手", action: "回答问题" })
// → "你是一个助手。请回答问题。"
```

## 12.5 课后练习

1. **添加新技能**：写一个 `code` 技能，专门回答编程问题
2. **模板文件**：把 Prompt 模板存到 `.md` 文件中读取
3. **技能组合**：让一个技能可以调用另一个技能的结果

# 📖 第 1 课：项目概述与环境准备

## 1.1 什么是 RAG？

**RAG = Retrieval-Augmented Generation（检索增强生成）**

通俗地说：**让 AI 在回答问题前，先去查资料。**

### 传统 AI 的问题
你问 ChatGPT："我们公司的请假流程是什么？"
ChatGPT 只能凭训练时的记忆回答——它根本不知道你们公司的规定！

### RAG 的解决方式
```
你问："我们公司的请假流程是什么？"

第一步（检索）：去公司的知识库里搜"请假流程"相关文档
第二步（增强）：把搜到的文档作为"参考材料"
第三步（生成）：AI 阅读参考材料后回答问题
```

### RAG 的核心流程

```
                    ┌───────────────────┐
                    │   你的知识库       │
                    │  (PostgreSQL +     │
                    │   pgvector)        │
                    └────────┬──────────┘
                             │
          ┌──────────────────┼──────────────────┐
          ▼                                     ▼
    ┌──────────┐                          ┌──────────┐
    │ 存入知识  │                          │ 提出问题  │
    │ 文本→向量 │                          │ 问题→向量 │
    └─────┬────┘                          └─────┬────┘
          │                                     │
          ▼                                     ▼
    ┌──────────┐                          ┌──────────┐
    │ 存入 PG  │                          │ 向量搜索 │
    │  (向量)  │                          │ 找相似内 │
    └──────────┘                          └─────┬────┘
                                                │
                                                ▼
                                          ┌──────────┐
                                          │ AI + 资料 │
                                          │  生成回答 │
                                          └──────────┘
```

## 1.2 本项目用到的技术

| 技术 | 作用 | 为什么选它 |
|------|------|-----------|
| **Next.js 14** | Web 框架 | 最流行的 React 全栈框架，App Router 简洁高效 |
| **TypeScript** | 编程语言 | 类型安全，开发体验好，减少 bug |
| **PostgreSQL** | 数据库 | 老牌关系型数据库，稳定可靠 |
| **pgvector** | PG 扩展 | 让 PostgreSQL 支持向量存储和相似度搜索 |
| **DeepSeek API** | AI 能力 | deepseek-embedding 做嵌入，deepseek-chat 做回答 |

## 1.3 环境准备清单

### 第一步：安装 Node.js

```bash
# 检查是否已安装
node --version    # 需要 >= 18
npm --version     # 需要 >= 9
```

如果没安装，去 https://nodejs.org 下载 LTS 版本。

### 第二步：安装 PostgreSQL

```bash
# macOS (Homebrew)
brew install postgresql@16

# 启动 PostgreSQL
brew services start postgresql@16

# 创建数据库
createdb rag_tutorial

# 验证
psql -d rag_tutorial -c "SELECT version();"
```

### 第三步：启用 pgvector 扩展

```bash
# 连接数据库
psql -d rag_tutorial

# 在 psql 中运行：
CREATE EXTENSION vector;

# 验证
SELECT extname, extversion FROM pg_extension WHERE extname = 'vector';
```

> **为什么需要 pgvector？**
> 普通的数据库只能存"文本"，不能比较"相似度"。
> pgvector 新增了一种 `vector` 数据类型，还提供 `<=>` 运算符来计算向量距离。
> 这就像给数据库装上了"相似度雷达"。

### 第四步：获取 DeepSeek API Key

1. 访问 https://platform.openai.com/api-keys
2. 点击 "Create new secret key"
3. 复制保存（key 只显示一次）
4. 确保账号有余额（充值链接：https://platform.openai.com/account/billing）

> **费用预估**：学完本教程大约需要不到 0.01 美元。

### 第五步：配置环境变量

在 `rag-tutorial/` 目录下创建 `.env.local` 文件：

```env
DEEPSEEK_API_KEY=sk-你的key在这里
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/rag_tutorial
```

> **为什么用 .env.local？**
> Next.js 会自动加载 `.env.local` 到 `process.env`。
> 这个文件已被加入 `.gitignore`，不会上传到 GitHub，保护密钥安全。

## 1.4 验证环境

```bash
# 进入项目目录
cd rag-tutorial

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

浏览器打开 http://localhost:3000，看到页面说明环境配置成功！

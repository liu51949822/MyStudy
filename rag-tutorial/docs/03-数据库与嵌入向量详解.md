# 📖 第 3 课：数据库与嵌入向量详解

## 3.1 为什么需要向量数据库？

### 传统数据库的局限

SQL 数据库擅长精确匹配：

```sql
SELECT * FROM documents WHERE content = '请假流程';
-- 只能找到完全匹配"请假流程"的文档
-- 找不到"请假规定"、"考勤制度"等相关内容
```

### 向量数据库的优势

```sql
SELECT content FROM documents
ORDER BY embedding <=> $1  -- 按"语义相似度"排序
LIMIT 3;
-- 能找到意思相近的所有内容
-- "请假流程" → "请假规定"、"考勤制度"、"休假政策"
```

> **一句话总结：**
> 传统数据库找"一模一样"，向量数据库找"意思相近"。

## 3.2 pgvector 详解

### 什么是 pgvector？

pgvector 是 PostgreSQL 的一个扩展，增加了：

1. **新的数据类型**：`vector(n)` — 存储 n 维向量
2. **距离运算符**：
   - `<=>` — 余弦距离（最常用）
   - `<#>` — 负内积距离
   - `<->` — L2 距离（欧几里得距离）

### 余弦距离的直觉理解

```
向量 A = [1, 0]  (指向正右方)
向量 B = [0, 1]  (指向正上方)
向量 C = [2, 0]  (指向正右方，更长)

A 和 C 的余弦距离 ≈ 0（方向相同，只是长度不同）
A 和 B 的余弦距离 > 0（方向不同）
```

> **余弦距离只关心方向，不关心长度。**
> 在文本语义中，"方向"代表"含义"，"长度"代表"细节多少"。
> 所以我们用余弦距离来判断语义相似度。

### 建表 SQL 拆解

```sql
CREATE TABLE IF NOT EXISTS documents (
  id SERIAL PRIMARY KEY,         -- 自增主键
  content TEXT NOT NULL,          -- 原始文本内容
  embedding vector(1536)          -- 文本的向量表示
);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `SERIAL PRIMARY KEY` | 自动递增，每条记录的唯一编号 |
| `content` | `TEXT NOT NULL` | 原始文本，可以为空吗？不行（NOT NULL） |
| `embedding` | `vector(1536)` | 1536 个浮点数，可以为空（刚开始可能还没算好向量） |

### 相似度搜索 SQL 拆解

```sql
SELECT
  id,
  content,
  embedding <=> $1::vector AS distance  -- 算余弦距离，命名为 distance
FROM documents                           -- 从 documents 表里查
ORDER BY distance ASC                    -- 按距离从小到大排（越近越相似）
LIMIT $2;                                -- 只返回前 K 条
```

> **$1 和 $2 是什么？**
> 这是参数化查询的占位符。`$1` 是传入的向量，`$2` 是 limit 值。
> 用参数化查询可以防止 SQL 注入攻击。

## 3.3 OpenAI Embedding API 详解

### 调用流程

```typescript
const response = await openai.embeddings.create({
  model: "deepseek-embedding",
  input: "要转换的文本",
});

// response.data[0].embedding 就是 1536 个数字
```

### 模型选择

| 模型 | 维度 | 每美元处理量 | 适用场景 |
|------|------|-------------|---------|
| deepseek-embedding | 1536 | 非常多 | ✅ 入门首选 |
| text-embedding-3-large | 3072 | 较少 | 需要更高精度时 |

> **为什么入门选 small？**
> - 成本低：同样的 1 美元可以处理更多文本
> - 速度快：计算量小
> - 维度低：数据库存储和计算更高效
> - 对入门教程来说精度完全够用

### API 调用例子

输入："今天天气真好"
输出（简化）：`[0.025, -0.613, 0.442, ...]`（共 1536 个）

输入："明天会下雨吗"
输出：`[0.031, -0.608, 0.435, ...]`（和上面的向量比较接近）

输入："如何安装 PostgreSQL"
输出：`[0.891, 0.234, -0.567, ...]`（和上面的向量差很远）

> **这就是 RAG 能工作的秘密！**
> 语义相似的文本 → 向量距离近
> 语义不同的文本 → 向量距离远

## 3.4 数据库连接字符串说明

```
DATABASE_URL=postgresql://用户名:密码@主机:端口/数据库名
```

参数拆解：
- `postgresql://` — 协议头，告诉程序用 PostgreSQL 协议连接
- `用户名` — 默认是 `postgres`
- `密码` — 安装时设置的密码，默认也是 `postgres`
- `主机` — 数据库在哪台机器上，本地就是 `localhost`
- `端口` — PostgreSQL 默认端口是 `5432`
- `数据库名` — 我们用 `rag_tutorial`

完整例子：
```
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/rag_tutorial
```

# 📖 第 8 课：文件上传处理

## 8.1 为什么需要文件上传？

在之前的课程中，我们只能在浏览器的文本框中手动粘贴文本。这在处理长文档时很不方便。本课我们增加文件上传功能，支持直接上传 `.txt` 和 `.md` 文件。

## 8.2 multipart/form-data

### 什么是 multipart/form-data？

HTTP 请求的 `Content-Type` 默认为 `application/x-www-form-urlencoded`，只适合传输简单的键值对。文件是二进制数据，需要一种特殊的编码方式。

**multipart/form-data** 就是把请求体分成多个"部件"，每个部件包含一个独立的数据段。

```
POST /api/upload HTTP/1.1
Content-Type: multipart/form-data; boundary=----Boundary123

------Boundary123
Content-Disposition: form-data; name="file"; filename="note.md"
Content-Type: text/markdown

这里是文件的内容...
可以有多行...

------Boundary123--
```

### 浏览器端的 FormData API

```typescript
const formData = new FormData();
formData.append("file", fileObject);

fetch("/api/upload", {
  method: "POST",
  body: formData,
  // 注意：不要手动设置 Content-Type！
  // 浏览器会自动设置，包括正确的 boundary
});
```

> **重要**：使用 FormData 时，**不要手动设置 `Content-Type` header**。浏览器会自动生成包括 `boundary` 的完整 Content-Type。如果手动设置，会丢失 boundary 信息导致服务端解析失败。

### 服务端解析

Next.js 的 App Router 提供了 `request.formData()` 方法：

```typescript
export async function POST(request: NextRequest) {
  const formData = await request.formData();
  const file = formData.get("file") as File;
  // ...
}
```

## 8.3 浏览器 File API

HTML5 引入了 File API，让 JavaScript 能直接读取用户选择的文件。

### 获取文件

```html
<input type="file" accept=".txt,.md" />
```

通过 `useRef` 获取文件对象：

```typescript
const fileInputRef = useRef<HTMLInputElement>(null);

// 在提交时
const file = fileInputRef.current?.files?.[0];
```

### File 对象的属性和方法

| 属性/方法 | 说明 | 示例 |
|-----------|------|------|
| `file.name` | 文件名 | `"note.md"` |
| `file.size` | 文件大小（字节） | `12345` |
| `file.type` | MIME 类型 | `"text/markdown"` |
| `file.text()` | 读取为文本 | `"# 标题\n内容..."` |

> **安全提醒**：`file.type` 可以被客户端伪造，不要仅依赖它做类型校验。应该同时检查文件名后缀和文件头（magic bytes）。

## 8.4 文件类型验证

### 白名单 vs 黑名单

- **黑名单**（不推荐）：禁止某些扩展名。攻击者总能找到新的扩展名。
- **白名单**（推荐）：只允许特定的扩展名。

```typescript
const filename = file.name.toLowerCase();
const allowedExtensions = [".txt", ".md"];
const isAllowed = allowedExtensions.some((ext) => filename.endsWith(ext));

if (!isAllowed) {
  return NextResponse.json(
    { error: "不支持的文件格式" },
    { status: 400 }
  );
}
```

### 文件大小限制

在生产环境中，还应该限制文件大小：

```typescript
const MAX_SIZE = 5 * 1024 * 1024; // 5MB
if (file.size > MAX_SIZE) {
  return NextResponse.json(
    { error: "文件过大，最大支持 5MB" },
    { status: 400 }
  );
}
```

## 8.5 完整上传流程

```
用户选择文件 → 点击上传
     │
     ▼
浏览器：new FormData() → append("file", file)
     │
     ▼
POST /api/upload (multipart/form-data)
     │
     ▼
服务端：request.formData() → 验证 → file.text()
     │
     ▼
调用 ingest(text)（分块 + 生成向量 + 存入数据库）
     │
     ▼
返回 { message: "✅ 已成功存入 5 个文本块！" }
```

# 📖 第 10 课：混合搜索 — 向量 + 关键词

## 10.1 为什么需要混合搜索？

### 纯向量搜索的弱点

向量搜索擅长"语义匹配"，但不是万能的：

```
搜索："工号 10086 的薪资是多少？"
向量搜索找到：↓
  0.92  "员工薪资计算方式"（意思接近，但没提工号）
  0.85  "薪资发放流程"（薪资相关，但没有具体信息）
  0.78  "考勤管理制度"（HR 相关，但跑题了）

理想结果：
  "工号 10086 的薪资明细"（精确匹配工号，但向量距离可能较远）
```

**问题**：向量搜索看不出来"10086"这个具体数字有多重要！

### 纯关键词搜索的弱点

关键词搜索擅长"精确匹配"，但也有问题：

```
搜索："请假流程"
关键词搜索找到：↓
  分数高  "请假流程"（完全匹配！）
  分数高  "请假审批流程"（匹配！）
  分数低  "年假申请方式"（没有"请假"这个词，但意思相关！）
```

**问题**：关键词搜索找不到"意思相近但用词不同"的内容。

### 混合搜索 = 两者结合

```
搜索："工号 10086 的薪资"
向量找到：
  "薪资计算方式"（意思接近）
  "员工薪酬体系"（意思接近）

关键词找到：
  "工号 10086 薪资记录"（精确匹配 10086！）

混合搜索（RRF 合并）：
  1. "工号 10086 薪资记录" ← 两者都匹配
  2. "薪资计算方式" ← 向量匹配
  3. "员工薪酬体系" ← 向量匹配
```

## 10.2 RRF 算法详解

RRF = Reciprocal Rank Fusion（倒数排序融合）

**核心思路**：不管什么搜索方式，结果按照"排名"来打分。

```
排名 1 → 分数 = 1/(60+1) = 0.0164
排名 2 → 分数 = 1/(60+2) = 0.0161
排名 3 → 分数 = 1/(60+3) = 0.0159
...
排名 10 → 分数 = 1/(60+10) = 0.0143
```

**合并过程**：

```
向量搜索结果（取前 10 条）：
  A: 0.0164  (排名 1)
  B: 0.0161  (排名 2)
  C: 0.0159  (排名 3)

关键词搜索结果（取前 10 条）：
  B: 0.0164  (排名 1)  ← B 在两个搜索结果里都出现了！
  D: 0.0161  (排名 2)
  E: 0.0159  (排名 3)

合并后：
  B: 0.0161 + 0.0164 = 0.0325  ← 两个搜索都排名靠前，总分最高！
  A: 0.0164 + 0 = 0.0164
  C: 0.0159 + 0 = 0.0159
  D: 0 + 0.0161 = 0.0161
  E: 0 + 0.0159 = 0.0159
```

**为什么取 60 作为常数？**
- RRF 中的常数（60）是经验值
- 它控制"排名重要性"的衰减速度
- 60 是比较常用的默认值

## 10.3 PostgreSQL 全文搜索实现

### tsvector — 文本→词向量

```sql
-- to_tsvector 把文本转换成"词向量"格式
SELECT to_tsvector('simple', 'RAG 是检索增强生成');
-- 结果：'rag':1 '检索':3 '增强':4 '生成':5 '是':2
--          ↑                 数字代表位置
--         词（已标准化）
```

### plainto_tsquery — 查询→查询向量

```sql
-- plainto_tsquery 把查询转换成"查询向量"
SELECT plainto_tsquery('simple', 'RAG 是什么');
-- 结果：'rag' & '是什么'
--         ↑ 用 & 连接（表示 AND）
```

### @@ 匹配操作符

```sql
-- @@ 检查 tsquery 是否匹配 tsvector
SELECT 'RAG 是检索增强生成'::tsvector @@ 'rag'::tsquery;
-- 结果：true

-- 用在 WHERE 子句中
SELECT content FROM documents
WHERE content_tsv @@ plainto_tsquery('simple', 'RAG');
-- 只返回包含"RAG"的文档
```

### ts_rank — 相关性排序

```sql
-- ts_rank 计算文本相关性分数
SELECT content, ts_rank(content_tsv, plainto_tsquery('simple', 'RAG')) AS score
FROM documents
WHERE content_tsv @@ plainto_tsquery('simple', 'RAG')
ORDER BY score DESC;
```

### GIN 索引

```sql
CREATE INDEX idx_documents_fts ON documents USING GIN(content_tsv);
```

> **GIN（Generalized Inverted Index）是什么？**
> 想象一本书最后的"关键词索引"：
> - RAG → 第 3, 15, 78 页
> - Embedding → 第 5, 12 页
> - Token → 第 8 页
>
> 这就是倒排索引！关键词 → 文档列表。
> 搜索"RAG"的时候直接去索引查，不用翻遍整本书。

## 10.4 混合搜索 SQL 拆解

```sql
WITH vector_results AS (
  -- 向量搜索：取前 10 条
  SELECT id, content, embedding <=> $1::vector AS vector_dist
  FROM documents
  ORDER BY vector_dist ASC
  LIMIT 10
),
keyword_results AS (
  -- 关键词搜索：取前 10 条
  SELECT id, content, ts_rank(content_tsv, plainto_tsquery('simple', $2)) AS kw_score
  FROM documents
  WHERE content_tsv @@ plainto_tsquery('simple', $2)
  ORDER BY kw_score DESC
  LIMIT 10
),
combined AS (
  -- 给每条结果算 RRF 分数
  SELECT id, content, 1.0 / (60.0 + ROW_NUMBER() OVER (...)) AS score
  FROM vector_results
  UNION ALL
  SELECT id, content, 1.0 / (60.0 + ROW_NUMBER() OVER (...)) AS score
  FROM keyword_results
)
-- 合并相同 ID 的分数（B 出现了两次，分数相加）
SELECT id, content, SUM(score) AS score
FROM combined
GROUP BY id, content
ORDER BY score DESC
LIMIT $3;
```

## 10.5 本章小结

| 搜索方式 | 擅长 | 不擅长 | 类比 |
|---------|------|--------|------|
| 向量搜索 | 语义匹配 | 精确匹配 | 理解意思的"人" |
| 关键词搜索 | 精确匹配 | 语义匹配 | 查字典的"机器" |
| 混合搜索 | 两者结合 | — | 人和机器合作 |

**关键公式**：
```
混合搜索 = 向量搜索 × RRF + 关键词搜索 × RRF
```
